# mp4tosrt

動画ファイルから自動で **SRT 字幕** を生成する Mac 用アプリ（Apple Silicon 専用）。

オープンソースの `ffmpeg` + `mlx-whisper`（Apple Silicon に最適化された Whisper 実装）を組み合わせ、複数ファイルを並列処理できます。さらにローカル LLM で日本語の誤認識を文脈に応じて校正したり、できあがった字幕を再エンコードなしで MP4 にミックスダウンしたりも可能。動画編集者の字幕テロップ作成や YouTuber の字幕入稿の下書きとしてどうぞ。

## できること

### 字幕生成

- **動画 → SRT 字幕の自動生成**（Whisper による多言語対応の音声認識）
- **複数ファイルの並列処理**（ffmpeg と mlx-whisper の同時実行数を個別に設定可能）
- **音声フィルタの自動適用**（ラウドネス正規化、ハイパス、ノイズ低減 ほか）
- **Whisper パラメータの調整**（無音判定閾値・対数確率閾値などを GUI スライダーで）

### 校正・補正

- **用語辞書による誤変換修正**（固有名詞の単純置換、`.txt` でメンテ可能）
- **ローカル LLM による日本語校正**（プライバシーを損なわず、Apple Silicon 上で完結）
- **共通プロンプト + ジョブごとの追加プロンプト**（動画の文脈を LLM に渡せる）
- **辞書を LLM 文脈に注入**（単純置換より柔軟な固有名詞補正）

### 出力・再実行

- **字幕埋め込み MP4 の出力**（`.subbed.mp4` として出力先フォルダに保存。再エンコードなし）
- **LLM 校正のみ再実行**（プロンプトを変えて何度でも試せる、Whisper はやり直さない）
- **MP4 字幕埋め込みのみ再実行**（`.corrected.srt` を手動で直してから埋め込み直せる）

### UX

- **ジョブ状態の自動保存**（アプリ再起動後も完了/プロンプト待ちジョブが復元される）
- **設定の自動保存**（出力先・モデル・フィルタ・LLM プロンプト等、すべて次回起動時に復元）
- **ライブログ表示**（ffmpeg / mlx_whisper / LLM 校正の進行状況をリアルタイムで確認）
- **アプリ内マニュアル**（このREADMEが画面右上「マニュアル」ボタンから読める）

## 動作環境

| 項目 | 必須 | 推奨 |
|---|---|---|
| **CPU** | Apple Silicon (M1 / M2 / M3 / M4 系) | M2 Pro 以上 |
| **macOS** | 13 (Ventura) 以降 | 14 (Sonoma) 以降 |
| **メモリ (RAM)** | 8GB | **16GB 以上**（LLM 校正を使う場合は事実上必須） |
| **空きストレージ** | 5GB | 10〜15GB（モデル複数試す場合は 20GB） |
| **ネットワーク** | 初回のみ必要（モデル DL に数 GB） | — |

### 機能ごとのメモリ目安

- **書き起こしのみ（Whisper large-v3-turbo）**: 8GB マシンで OK
- **書き起こし + LLM 校正（Qwen3.5 4B）**: 16GB 推奨
- **書き起こし + LLM 校正（Qwen3 8B）**: 16GB 必須・24GB 以上推奨

### ストレージ目安

- アプリ本体: 約 150MB
- Whisper モデルキャッシュ（自動DL、`~/.cache/huggingface/`）: モデル1つあたり 1〜3GB
- LLM モデル（任意DL、`~/Library/Application Support/mp4tosrt/models/`）: 1モデル 2.5〜5GB

### 非対応環境

⚠️ **Intel Mac / Linux / Windows は非対応**（`mlx-whisper` が Apple Silicon 専用のため）。  
⚠️ macOS 12 以前は mlx フレームワークの動作対象外。

# 一般ユーザー向け（動画編集者向け）

## インストール手順

### Step 1: アプリ本体をダウンロード

1. [GitHub Releases](https://github.com/heavymoons/mp4tosrt/releases) ページから最新版の `mp4tosrt-x.x.x-arm64.dmg` をダウンロード
2. ダウンロードした `.dmg` をダブルクリックでマウント
3. `mp4tosrt.app` を **Applications フォルダ** にドラッグ＆ドロップ

### Step 2: 隔離属性を外す（未署名アプリのため一回だけ必要）

このアプリは **未署名（無料配布）** なので、macOS のセキュリティ保護を一度だけ手動で解除する必要があります。

**ターミナル.app** を開いて、次の1行をコピペして実行してください：

```bash
xattr -d com.apple.quarantine /Applications/mp4tosrt.app
```

> ターミナルが開きにくい場合は、`Applications/mp4tosrt.app` を Finder で **右クリック → 開く → 開く** でも代替できます（初回のみ。次からは普通にダブルクリックで開きます）。

### Step 3: 必要ツールのインストール

このアプリは内部で `ffmpeg`（音声変換）と `mlx-whisper`（音声認識）という 2 つのコマンドラインツールを使います。**ターミナルで一度だけ** 以下を実行してインストールしてください。

#### 3-1. Homebrew（パッケージマネージャ）

すでに `brew` コマンドが使えるならスキップ。`brew --version` でバージョンが表示されない場合のみ、以下を実行：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

インストール完了時にターミナルに表示される `eval "$(/opt/homebrew/bin/brew shellenv)"` のような行を、案内に従って `~/.zprofile` に追加してください（その後ターミナルを開き直す）。

#### 3-2. ffmpeg

```bash
brew install ffmpeg
```

#### 3-3. mlx-whisper

```bash
brew install pipx
pipx ensurepath
pipx install mlx-whisper
```

`pipx ensurepath` 実行後はターミナルを **一度閉じて開き直して** ください。

### Step 4: 起動と環境チェック

1. Launchpad または Applications から **mp4tosrt** を起動
2. 画面上部の「環境チェック」で `ffmpeg` / `mlx_whisper` の両方が **緑のチェック** で表示されれば準備完了
3. 赤の場合はインストールコマンドのコピーボタンが出るので、それをターミナルに貼って実行

## 基本の使い方

1. **出力先フォルダを指定** — 設定の「出力先」で SRT を保存する場所を決める（次回起動時に復元される）
2. **動画ファイルをドロップ** — 画面中央のドロップゾーンに動画ファイルをドラッグ＆ドロップ（複数同時OK）
3. **進行を待つ** — ジョブごとに `音声変換中` → `文字起こし中` → `完了` と進む
4. **字幕ファイルが完成** — 完了したら「SRT を表示」ボタンで Finder で開ける

出力ファイル名は `<元の動画ファイル名>.srt`（例: `interview.mp4` → `interview.srt`）。

> 画面右上の **「マニュアル」** ボタンでこの README をアプリ内に表示できます。

### ジョブごとに使えるアクション

各ジョブ行の右側にあるボタン群：

| ボタン | 説明 |
|---|---|
| **SRT を表示** | 生成された SRT を Finder で表示（完了後に出現） |
| **MP4 へ字幕埋め込み** | `.corrected.srt` (or `.srt`) を元に MP4 を作り直す。**SRT を手動編集後に押すと反映される** |
| **+ 追加プロンプト** / **✎ 追加プロンプト** | LLM 校正向けの動画固有の文脈を入力（後述） |
| **ログ** | ffmpeg / mlx_whisper / LLM の生ログを展開表示 |
| **中止** | 実行中のジョブをキャンセル |
| **削除** | 完了/エラーのジョブをリストから消す |
| **LLM校正からやり直す** / **LLM校正を開始** | LLM 校正以降のステップだけ再実行（追加プロンプト編集後など） |

「**完了済みを片付け**」ボタン（ジョブリスト右上）で、終わった/エラーのジョブを一括削除できます。

## 設定項目

### 基本

| 項目 | 説明 |
|---|---|
| 出力先 | SRT を保存するフォルダ |
| 字幕を埋め込んだ `.subbed.mp4` も出力 | ON にすると、書き起こし完了後に元動画＋SRT を ffmpeg でミックスして MP4 を生成。再エンコードなしで字幕トラック付き動画ができる。元動画コーデックが mp4 互換 (h264/h265/aac等) でない場合は失敗するが SRT 生成自体は影響なし |
| Whisper モデル | デフォは `mlx-community/whisper-large-v3-turbo`（バランス重視）。日本語特化なら `kaiinui/kotoba-whisper-v2.0-mlx` |
| 言語 | 自動検出 / 日本語 / 英語など。明示すると精度が上がることがある |
| ffmpeg 並列 | 音声変換の同時実行数（デフォ 2） |
| mlx-whisper 並列 | 文字起こしの同時実行数（デフォ 1。GPU メモリを多く使うので増やしすぎ注意） |

### 詳細設定 — 音声フィルタ

素材に応じて on/off を切り替える。**デフォ ON のものは副作用がほぼ無い保守的な設定**。

| フィルタ | デフォ | 効くケース |
|---|---|---|
| ラウドネス正規化（loudnorm） | ON | 録音レベルがバラバラなとき。一番リターン大きい |
| ハイパスフィルタ | 80 Hz | エアコン・空調などの低域ノイズ。男性の低音を残すなら 80、強めに切るなら 120/200 |
| コンプレッサ | OFF | 話者距離が違うインタビューなど、音量差が激しい素材 |
| ノイズ低減（afftdn） | OFF | 強いノイズ素材のみ。アーティファクトで認識が悪化することもあるので A/B 推奨 |

詳細：

- **loudnorm + highpass=80** はほぼ全素材で安全。
- **denoise（afftdn / rnnnoise 系）は要注意**。Whisper 自体がノイズに対してそこそこロバストなので、整音アーティファクトが逆効果になることがある。特に音楽が混ざる素材で声まで削れる。
- **コンプは loudnorm より前段** に入れている（自然な動作になる）。
- EQ で「人間に聞き取りやすくする」処理は Whisper には効かないか悪化する印象なので入れていない。

### 詳細設定 — Whisper オプション

| 項目 | デフォ | 説明 |
|---|---|---|
| 前文脈を考慮 | OFF | `--condition-on-previous-text` 相当。ON にすると「！」連発などのリピートループにハマることがあるのでデフォは OFF。固有名詞の表記一貫性を優先したい短尺素材だけ ON 推奨 |
| no-speech-threshold | 0.30 | そのセグメントを「無音」と判定する閾値。Whisper のデフォ (0.6) は日本語に対して厳しすぎて「はっきり喋ってる場所が抜ける」現象を起こしやすいので 0.3 に緩めている。下げるほど抜けが減るが、ノイズを発話と誤認する可能性も上がる |
| logprob-threshold | -1.50 | デコード信頼性の下限。下回ったセグメントは「失敗」扱いで空にされる。Whisper のデフォ (-1.0) より緩めの -1.5 で抜けを減らす |

### 詳細設定 — 後処理（ハルシネーション抑制）

Whisper は無音区間や音楽パートで「**ご視聴ありがとうございました**」のような訓練データ由来のフレーズを誤って吐く（ハルシネーション）ことが知られています。これを SRT 生成後にキュー単位で削除する機能。**デフォルト ON**。

#### デフォルトで抑制されるパターン

YouTube 動画でほぼ確実にハルシネーションと判別できるものだけを内蔵：

- ご視聴ありがとうございました / ご視聴ありがとうございます
- ご清聴ありがとうございました / ご清聴ありがとうございます
- チャンネル登録お願いします / 高評価チャンネル登録お願いします
- バイバイ
- 字幕 by

正規化（句読点・記号を無視）した上で **キュー全体のテキスト** がパターンと一致したキューだけを削除します（部分一致では削除しない）。例えば「実はご視聴ありがとうございました、と言いたいわけです」のように本文中に含まれる場合は残ります。

#### 追加パターンの登録

設定の **「ハルシネーション抑制」** セクション内で **「編集」** ボタンを押すと、デフォルトの `hallucinations.txt` をその場で編集できます。1行1パターンで自分の動画特有のハルシネーションを追加してください。

```
# 自分の動画でよく出るハルシネーション
ごめん
おやすみ
うん
バイバイまた来週
```

ルール：
- 1 行 1 パターン
- `#` で始まる行はコメント
- 空行は無視

「**別のファイル…**」で任意のテキストファイルを指定することも可能。プロジェクトごとに違うリストを使い分けたい場合に。「**デフォルトに戻す**」でアプリのデフォルトファイル（`~/Library/Application Support/mp4tosrt/hallucinations.txt`）に戻せます。

#### 注意点

- 「ごめん」「うん」など短い言葉は **本物の発話との誤削除リスク** があるので、デフォルトには含めていない。自分の動画で頻出するなら追加リスト側で対応
- 本物の発話で「ご視聴ありがとうございました」を喋っている動画（解説者の挨拶等）では誤削除されるので、その場合は設定で OFF にする
- LLM 校正の前に実行されるので、抑制された内容は LLM 入力にも含まれない

### 詳細設定 — 後処理（用語辞書 / 単純置換）

固有名詞・専門用語の誤変換を、SRT 生成後に一括置換する。

設定の **「用語辞書」** セクション内で **「編集」** ボタンを押すと、デフォルトの `dictionary.txt`（`~/Library/Application Support/mp4tosrt/dictionary.txt`）をその場で編集できます。「**別のファイル…**」で別ファイル選択、「**デフォルトに戻す**」でデフォルトファイルに切替も可能。

#### 辞書ファイルの書き方

タブ区切りのテキストファイル（`.txt` / `.tsv` / `.dict`）。

```
# 政治家
タカイチ	高市
イシバ	石破

# 軍事用語
ステルスファイター	ステルス戦闘機

# 自分のプロジェクト固有名詞
ヘビームーンズ	heavymoons
```

ルール：

- 1 行 1 ルール。`誤変換<TAB>正解` の形式
- `#` で始まる行はコメント
- 空行は無視
- 区切り文字は **タブ文字**（スペースではない）
- 単純な文字列置換（正規表現ではない）

#### 運用のコツ

- プロジェクト・ジャンルごとに辞書を分ける（政治系 / 技術系 / 自社固有名詞）
- 短すぎる文字列は避ける（誤適用のリスク。例えば `AI` を `エーアイ` に変換するルールは入れない方がいい）
- 何度も同じ間違い方をする単語だけを溜めていく
- 完成された辞書は「資産」として残し、案件ごとに切り替える

### 詳細設定 — 後処理（LLM 校正 / ローカル）

ローカル LLM（Qwen 系）で字幕全体を読ませて、Whisper の崩れた日本語を自然な表現に整える機能。**書き起こし完了後の `<出力先>/<元動画名>.srt` を読み込み → 校正 → `<出力先>/<元動画名>.corrected.srt` として並べて保存** する。元の SRT は触らない。**ネット送信なし、すべてローカル処理**（プライバシー保護）。

> 以降このドキュメントでの `<元動画>.srt` などの表記は **「出力先フォルダ」配下の `<元動画ファイル名（拡張子除く）>.srt`** を意味します。元動画ファイルのある場所とは別物です（出力先フォルダは設定で指定）。

#### セットアップ

`node-llama-cpp` がアプリにバンドルされているので **追加のインストール作業は不要**。設定画面の「詳細設定」→「後処理 — LLM 校正」セクションで：

1. 「LLM 校正を有効化」をオン
2. モデルを選ぶ（デフォは Qwen3.5 4B Q4_K_M、約 2.5GB）
3. 「ダウンロード」ボタンを押す
4. 進捗バーが消えたら準備完了

モデルは `~/Library/Application Support/mp4tosrt/models/` に保存される。

#### モデル選択肢

| モデル | サイズ | 特徴 |
|---|---|---|
| **Qwen3.5 4B Q4_K_M（デフォ）** | 約 2.5GB | 字幕校正に必要十分、速い、メモリ少なめ |
| Qwen3.5 4B Q5_K_M | 約 3GB | 4B でやや高品質 |
| Qwen3.5 9B Q4_K_M | 約 5.5GB | より高精度、固有名詞補正が強い |
| Qwen3.5 9B Q5_K_M | 約 6.5GB | 9B でさらに高品質 |
| Gemma 4 E2B Q4_K_M | 約 1.5GB | Google系、軽量・高速 |
| Gemma 4 E2B Q5_K_M | 約 1.8GB | Gemma 軽量・やや高品質 |
| Gemma 4 E4B Q4_K_M | 約 2.5GB | Google系、Qwen3.5 4B と比較対象 |
| Gemma 4 E4B Q5_K_M | 約 3GB | Gemma E4B やや高品質 |

**選び方の目安:**
- 16GB Mac → Qwen3.5 4B / Gemma 4 E4B（4Bクラス）
- 32GB+ Mac → Qwen3.5 9B（より高精度）
- 日本語の固有名詞補正は **Qwen 系がやや強い** 印象（個人観測）。気になる人は Gemma との A/B 比較を

#### 共通プロンプト

設定の「LLM 校正」セクションにある共通プロンプトは、**全ジョブに共通でLLMに渡される文脈情報**。例：

```
自分のYouTubeチャンネル「heavymoons」の動画です。日本語の固有名詞や軍事/政治用語が頻出します。
```

#### ジョブごとの追加プロンプト

各ジョブ行の **「+ 追加プロンプト」** ボタンでテキストエリアが開き、その動画固有の文脈を LLM に渡せる。

例：
- `〇〇による政治系ライブ配信の書き起こし。日本の政治家・地域名が頻出する`
- `IT勉強会の発表。React や TypeScript などの技術用語が多い`
- `インタビュー番組。話者は2名、フォーマルな文体で校正してほしい`

入力後、テキストエリアからフォーカスを外すと自動保存。共通プロンプトとあわせて LLM に渡される。

「**ジョブごとの追加プロンプトを必須にする**」設定（デフォ ON）が有効な場合、Whisper の書き起こしが終わると、ジョブは「**プロンプト入力待ち**」状態（オレンジ色）で一時停止し、`<元動画>.srt` だけ生成された状態で待機する。プロンプト入力後、**「LLM校正を開始」** ボタンで LLM 校正と字幕埋め込みが進行する。

設定オフにすると、追加プロンプトが空でも自動で LLM 校正が走る。

辞書 = ピンポイントの誤変換修正、追加プロンプト = 動画全体の文脈・スタイル指示、と使い分けると効果的。

#### 用語辞書を LLM に渡す

「用語辞書をプロンプトに渡す」を **オン** にすると、上記の用語辞書ファイル（誤変換 → 正解）を **LLM のシステムプロンプトに埋め込む**。これにより：

- 単純置換と違って **文脈に応じた適用** が可能（同じ表記でも文脈が違えば置換しない）
- 辞書に載ってない類似の誤変換も推論で修正される
- 短すぎるルールでの誤適用が起きにくい

辞書ファイルは「単純置換」と共通。一つの辞書ファイルを単純置換だけ・LLM だけ・両方に使うか選べる。

#### LLM 校正からやり直す

ジョブが完了済みでも、追加プロンプトの編集後などに **「LLM校正からやり直す」** ボタンで LLM 校正以降のステップだけ再実行できる（ffmpeg 変換と Whisper はスキップ）。

用途：
- 追加プロンプトを書き換えて何度も試したい
- モデルを切り替えてみたい（設定で変更後にやり直し）
- 字幕埋め込み MP4 だけ生成したい / し直したい
- 用語辞書を更新したので再適用したい

`<元動画>.corrected.srt` と `<元動画>.subbed.mp4` は上書きされるが、ベースの `<元動画>.srt` には触らない。Whisper の重い処理を省けるので、プロンプトの試行錯誤が現実的なコストでできる。

#### MP4 へ字幕埋め込みのみやり直す

ジョブの **「MP4 へ字幕埋め込み」** ボタンで、字幕埋め込みステップだけを単独で再実行できる。LLM 校正もスキップする。

用途：
- 自動生成された `<元動画>.corrected.srt` を **テキストエディタで手直しした後**、その内容を反映した `<元動画>.subbed.mp4` を作りたい
- 「字幕を埋め込んだ MP4 も出力」を後からオンにして、過去のジョブにも適用したい
- 一度だけ手動で MP4 出力したい（設定の常時 ON にはせず）

ボタンを押すと、現在ディスクにある SRT を読み込んで ffmpeg で再ミックス：
- `<元動画>.corrected.srt` があればそれを優先使用
- 無ければ `<元動画>.srt`（手で編集している場合はこちらも反映される）

ベースの SRT / corrected SRT は触らず、`<元動画>.subbed.mp4` だけ上書きされる。

#### 注意点

- 初回実行時はモデルを RAM にロードするため数十秒かかる（以降は常駐するので速い）
- 1 時間動画でだいたい数分〜十数分（モデル・マシン性能による）
- LLM 出力が形式不一致の場合は元のキューを保持（タイムスタンプは絶対に壊さない設計）
- メモリ使用量: 4B Q4 で約 4GB、8B Q4 で約 6GB のRAMを占有

## 出力ファイル

入力動画 `interview.mp4` に対して、以下が出力先フォルダに生成される：

| ファイル | 出力条件 |
|---|---|
| `interview.srt` | 常に生成（標準 SRT、UTF-8） |
| `interview.corrected.srt` | LLM 校正を有効にした場合 |
| `interview.subbed.mp4` | 「字幕を埋め込んだMP4も出力」を有効にした場合（再エンコードなし、LLM 校正済み SRT があれば優先使用） |

元動画は触らない。

## 認識精度を上げるには（優先度順）

1. **モデルを `kaiinui/kotoba-whisper-v2.0-mlx` に**（日本語特化、large-v3 ベースで日本語ベンチで本家超え）
2. 音量がバラバラなら **loudnorm を確認**（デフォ ON だが無効化されていないか）
3. 音量差が激しいインタビュー素材なら **コンプレッサを ON**
4. 同じ単語の誤変換が繰り返すなら **用語辞書** を整備
5. 長尺で誤認識が連鎖するなら **前文脈を考慮を OFF**
6. 「はっきり喋ってる場所が抜ける」場合は **no-speech-threshold を下げる**（0.3 → 0.1 など）
7. それでも崩れた日本語が多いなら **LLM 校正** を有効化（`.corrected.srt` を生成）

## トラブルシューティング

### `mlx_whisper` が「未インストール」のまま

`pipx install mlx-whisper` 後にターミナルを開き直すか `pipx ensurepath` を実行。アプリも再起動してください。アプリは PATH に以下を補完しているので、これらの場所にあれば検出できる：

- `/opt/homebrew/bin`
- `/usr/local/bin`
- `~/.local/bin`
- `~/Library/Python/3.11/bin` ～ `3.13/bin`

### 文字起こしが遅い / メモリ不足になる

- `mlx-whisper 並列` を 1 に下げる
- より小さいモデル（`whisper-medium-mlx` / `whisper-small-mlx`）に切り替える
- `whisper-large-v3-turbo` は精度と速度のバランスが良いのでデフォルト

### 「開発元を確認できないため開けません」と表示される

未署名アプリの初回起動時のメッセージ。Step 2 の `xattr` コマンドを実行してください。すでに実行済みなのに出る場合は、Finder で `mp4tosrt.app` を **右クリック → 開く → 開く** で初回バイパス。

### LLM 校正が効かない（書き起こしほぼそのまま）

- 4B モデルは限界がある場合がある。**Qwen3 8B** に切り替えてみる
- 追加プロンプトを **より具体的に** 書く（「動画の概要」だけでなく、「この用語が出るはず」「こう誤認識されがち」を明記）
- 「LLM校正からやり直す」で何度か試行錯誤

## 既知の制限

- macOS（Apple Silicon）専用。Intel Mac / Linux / Windows は非対応
- VAD 前処理（silero-vad）は未実装。長い無音区間で「ご視聴ありがとうございました」型のハルシネーションが出る場合は、用語辞書側で消すか、トリミングしてから入力する
- プリセット保存機能は未実装

# 開発者向け

## 開発環境

```bash
git clone https://github.com/heavymoons/mp4tosrt.git
cd mp4tosrt
npm install
npm run dev          # 開発モード（ホットリロード）
```

主要コマンド：

```bash
npm run build        # 本番ビルド（バンドルのみ、.app は作らない）
npm run typecheck    # TypeScript 型チェック
npm run dist         # 配布用 .dmg / .zip を /tmp/mp4tosrt-release/ に出力
```

## トラブルシューティング（開発時）

### `npm run dev` で `Cannot read properties of undefined (reading 'whenReady')` が出る

VS Code から起動した場合、環境変数 `ELECTRON_RUN_AS_NODE=1` が継承されることがある。`package.json` の `dev` スクリプトでは `env -u ELECTRON_RUN_AS_NODE` で除外している。手動で起動する場合は同様に外すこと。

## リリース手順

1. `package.json` の `version` を上げる
2. `git tag vX.Y.Z && git push --tags`
3. GitHub Actions が自動で macos-14 (Apple Silicon) 上でビルドし、`.dmg` / `.zip` を Release にアップロードする
4. ローカルから手動でも可: `npm run dist` → `gh release create vX.Y.Z /tmp/mp4tosrt-release/*.dmg /tmp/mp4tosrt-release/*.zip`

ビルドは ad-hoc 署名のみ（無料）。Apple Developer Program (年$99) で本署名 + ノータリゼーションする場合は、`mac.identity` を実 Developer ID に変更し、`afterSign` で notarytool を呼ぶ。
